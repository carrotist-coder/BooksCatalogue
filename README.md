# Books Catalogue (The Library App)

A modern web-based book discovery application powered by the Open Library API. Users can search for millions of books and manage a personal favorites list with persistent storage.
### [Live Demo](https://carrotist-coder.github.io/BooksCatalogue/)

## 2. How to run the app

### Prerequisites
- [Node.js](https://nodejs.org/) (v16 or higher recommended)
- [npm](https://www.npmjs.com/) (comes with Node.js)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/carrotist-coder/BooksCatalogue

2. Navigate to the project folder:
   ```bash
   cd books-catalogue

3. Install dependencies:
   ```bash
   npm install
   ```

### Development
To run the application in development mode with a local server:
```bash
npm run start
```
The app will be available at http://localhost:3000.

### Build for Production
To generate a production-ready build (minified and optimized) in the `dist` folder:
```bash
npm run build
```

## 3. Project Structure

The project follows a modular architecture to ensure a clear separation of concerns and high maintainability:

- **`dist/`**: The distribution folder containing production-ready bundled files generated by Webpack.
- **`src/`**: The core source code directory.
   - **`assets/`**: Static assets - SVG icons for the UI (e.g., heart icons).
   - **`scripts/`**: Modular JavaScript files:
      - `api.js`: Communication with the Open Library API.
      - `storage.js`: LocalStorage operations for saving and retrieving favorite books.
      - `theme.js`: Logic for detecting, initializing, and toggling between Light and Dark themes.
      - `ui.js`: Reusable functions for generating HTML templates and rendering UI components (cards, messages, loaders).
   - **`styles/`**: CSS files.
   - **`utils/`**: Utility modules and global configurations:
      - `consts.js`: Centralized application constants.
      - `debounce.js`: Implementation of the debounce pattern to optimize search input performance.
      - `utils.js`: Shared helper functions for data formatting and URL construction.
   - **`index.js`**: The central entry point.
- **`index.html`**: The main application skeleton.
- **`package.json`**: Defines project dependencies and build scripts (start, build).
- **`webpack.config.js`**: Configuration for the Webpack bundler.
